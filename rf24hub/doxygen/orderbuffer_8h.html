class  OrderBuffer
 

Detailed Description

Der Orderbuffer ist ein Objekt zur Speicherung von Einzelanforderungen.

Kommt eine Anforderung im Hub an, wird diese zunächst im Orderbuffer gespeichert. Besteht Kontakt zu einem Node (bei ankommendem Heartbeat) werden alle offenen Orderbuffereinträge für diesen Node zu einer Orderstruktur aggregiert. In einer Order können 6 Orderbuffereinträge zum Node übertragen werden.
